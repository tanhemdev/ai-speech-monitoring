## Problem

Patients with neurological conditions (ALS, Parkinson's, MS, stroke recovery) lose the ability to self-monitor speech deterioration between infrequent clinical visits. By the time a clinician detects regression, weeks of intervention time have been lost. Existing telehealth tools treat speech as binary (can talk / can't talk) rather than tracking the gradient of deterioration.

## Solution

Designed a speech monitoring platform that uses AI to track speech quality between clinical visits, surfacing early deterioration signals to both patients and clinicians.

**What I did:**

- Conducted **UX research** directly with neurologically affected patients — understanding daily routines, cognitive load, and emotional context around speech loss
- Translated research findings into a **full PRD** with user flows, edge cases, and accessibility requirements
- Designed **wireframes in Figma** for clinical team review and developer handoff
- Created **adaptive UI states** based on detected fatigue levels — the interface simplifies when the patient is struggling, not a static one-size-fits-all layout

## PRD Scope

| Section | Detail |
|---|---|
| **User personas** | Patient (early-stage), patient (advanced), caregiver, clinician |
| **Core flows** | Daily speech check-in, deterioration alert, clinician dashboard review |
| **Edge cases** | Patient fatigue mid-session, ambient noise, emotional speech vs. deterioration |
| **Accessibility** | WCAG compliant, large touch targets, voice-first navigation, high contrast |
| **Adaptive UI** | Interface complexity reduces based on detected cognitive/physical fatigue |
| **Data model** | Speech quality metrics, session history, trend visualization for clinicians |

## Key Insight

The breakthrough from user research: patients don't fail to monitor their speech because they're lazy — they fail because the monitoring itself is exhausting. The UI must **adapt to the patient's current state**, not demand a fixed level of interaction regardless of how they're feeling that day.

## Deliverables

- Complete PRD (problem, personas, flows, edge cases, technical requirements)
- Figma wireframes for patient and clinician interfaces
- Accessibility audit and WCAG compliance checklist
- Full dev handoff documentation — 100% ready for engineering
